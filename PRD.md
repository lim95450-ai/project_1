2. MVP 개발 방향

추천 MVP 방식

처음에는 QR 안에 HTML 전체를 넣지 말고, 웹앱 URL + id만 넣는 방식으로 테스트하세요.



QR 내용:

https://script.google.com/macros/s/배포ID/exec?sid=abc123



휴대폰으로 QR을 찍으면 웹앱이 열리고, sid=abc123에 해당하는 근무표 데이터를 구글시트에서 읽어서 보여줍니다.

이 방식이 1차 테스트에 좋은 이유는:





QR 용량 문제가 거의 없음



휴대폰에서 바로 보기 좋음



HTML 디자인을 마음껏 넣을 수 있음



데이터 수정 후 다시 열면 최신 내용 반영 가능



QR이 아주 단순해서 스캔 성공률이 높음

단점은 외부망에서 구글 웹앱 접근이 가능해야 한다는 것입니다. 폐쇄망 반출 테스트가 아니라 “아이디어 검증”용으로는 이게 베스트입니다.

지금은 A안으로 테스트하고, 가능성이 보이면 B안으로 확장하는 게 좋습니다.

# 프로젝트 지침



## 목적

Google Sheets + Apps Script 기반의 근무표 QR 공유 웹앱을 만든다.

근무자, 휴가, 말뚝근무, 근무시간, 필요 인원을 반영해 하루치 근무표를 생성하고, 휴대폰에서 보기 좋은 HTML 화면으로 공유한다.



## 핵심 요구사항

1. 구글시트를 데이터베이스처럼 사용한다.

2. 관리자 화면에서 근무표를 생성한다.

3. 생성된 근무표는 고유 sid를 가진다.

4. QR 코드는 `/exec?sid=...` 형태의 조회 URL을 담는다.

5. 조회 화면은 모바일에서 보기 좋게 반응형으로 만든다.

6. 휴가자는 붉은 계열 배지로 표시한다.

7. 말뚝근무자는 파란 계열 배지로 표시한다.

8. 한 시간대에 같은 사람이 중복 배치되지 않게 한다.

9. 모든 주요 함수에는 짧은 주석을 단다.

10. 민감정보를 다루지 않도록 이름은 테스트 단계에서 가명으로 사용한다.



## 시트 구조

- 설정

- 근무자

- 휴가

- 말뚝근무

- 생성결과



## 우선순위

1. 시트 초기화 함수

2. 샘플 데이터 입력 함수

3. 근무표 생성 함수

4. 관리자 HTML 화면

5. 조회 HTML 화면

6. QR 생성

7. README 작성



## 개발 방식

작은 단위로 구현하고, 각 단계마다 테스트 가능한 상태를 유지한다.

기능이 많아지더라도 Code.gs가 너무 비대해지지 않도록 역할별 함수로 분리한다.



## 금지

- 외부 서버에 근무 데이터를 전송하지 않는다.

- QR 생성 API에 실제 근무 데이터 전체를 보내지 않는다.

- 초기 버전에서 복잡한 최적화 알고리즘을 넣지 않는다.

- 대용량/장기 근무표를 한 번에 처리하려고 하지 않는다.



9. Codex에게 첫 작업으로 시킬 프롬프트

이걸 그대로 넣으면 됩니다.



Google Apps Script 프로젝트를 만들려고 합니다.



목표:

Google Sheets에 입력된 근무자, 휴가자, 말뚝근무, 근무시간, 필요 인원을 읽어서 하루치 야간 순번 근무표를 생성하고, 모바일에서 보기 좋은 HTML 조회 화면과 QR 공유 기능을 만드는 것입니다.



먼저 1차 MVP를 구현해주세요.



요구사항:

1. Code.gs, admin.html, view.html, style.html, script.html 구조로 작성

2. setupSheets() 함수로 필요한 시트와 헤더를 자동 생성

3. seedSampleData() 함수로 테스트용 샘플 데이터 입력

4. generateSchedule() 함수로 하루치 근무표 생성

5. 생성 결과는 생성결과 시트에 sid, 생성일시, 날짜, 제목, json, html 필드로 저장

6. doGet(e)는 mode=admin이면 관리자 화면, sid가 있으면 조회 화면을 반환

7. 관리자 화면에는 “시트 초기화”, “샘플 데이터 입력”, “근무표 생성”, “QR 생성/링크 복사” 버튼 제공

8. 조회 화면은 모바일 반응형 카드/표 UI로 표시

9. QR에는 HTML 전체가 아니라 조회 URL만 담기

10. QR 생성은 클라이언트에서 처리하되, 외부 API로 실제 근무 데이터를 보내지 않기

11. qrcode.js 같은 라이브러리를 사용할 경우 별도 파일 또는 인라인 코드로 관리

12. Apps Script HTML Service 환경에서 동작하도록 google.script.run을 사용

13. 코드에는 초보자가 이해할 수 있는 주석을 달기

14. README.md에 배포 방법과 사용 순서를 작성



주의:

초기 버전에서는 복잡한 최적화보다 동작하는 MVP를 우선합니다.

Apps Script 웹앱은 배포 설정에 따라 접근 범위가 달라집니다. 웹앱은 doGet(e) 또는 doPost(e)를 통해 브라우저에서 UI를 제공할 수 있고, 배포 시 접근 권한을 정해야 합니다.

12. 처음에는 이렇게 판단하면 됩니다

테스트용으로는

QR = 웹앱 조회 URL





이게 맞습니다.



폐쇄망 반출 실험용으로는

QR = 압축된 근무표 데이터





이건 나중입니다.



실사용 근무 공유용으로는

QR = 조회 URL + sid





이게 제일 편합니다.
